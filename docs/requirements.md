| 版 | 内容 |

## 1. 背景と目的

LAB-1 は減算合成の信号経路(OSC → FILTER → AMP ENV)を学ぶためのブラウザシンセである。
v2.0 では「同じ合成プリミティブの組み合わせでドラム音も旋律も作れる」ことを軸に、
**音作り(SOUND)** と **リズム構築(PATTERN)** を分離した 2 タブ構成とする。

### 1.1 プロジェクトゴール(優先順)

| # | ゴール | 含意 |
|---|--------|------|
| G1 | 合成の仕組みの学習 | 音源・エフェクトのコードは信号経路が読める構造であること。ブラックボックス禁止 |
| G2 | 実際の曲作りで使える | 演奏レイテンシ・音質・保存機能が実用水準であること |
| G3 | OSS / ポートフォリオ公開 | 依存最小・コード可読性・ドキュメント整備。「プリミティブから音とエフェクトを組み立てる」を題材として明確に打ち出す |

### 1.2 スコープ外(明示)

- サンプル再生(Sampler / Player 系)。合成のみで音を作る
- MIDI 入出力(将来検討)
- オーディオファイル書き出し(将来検討。JSON エクスポートは対象内)
- 同一プリセットのポリフォニック演奏(SOUND はモノフォニック)

---

## 2. 全体構成

| タブ | 役割 |
|------|------|
| **SOUND** | 単体の音作り。1 ボイス(モノシンセ + FX)を設計し、名前付きプリセットとして保存する |
| **PATTERN** | SOUND で作ったプリセットを各トラックに割り当て、16 ステップでリズム/フレーズを組む |

---

## 3. 機能要件

### 3.1 共通(FR-1xx)

- **FR-110** SOUND / PATTERN のタブ切替を提供する。切替はワンタップ。
- **FR-111** タブ切替時に再生中の音は自然に停止する(リリース経由。ブツ切りにしない)。PATTERN 再生中に切り替えた場合はシーケンサーも停止する。
- **FR-112** 両タブは共通のマスター出力(音量・アナライザー)を共有する。オシロスコープはどちらのタブでも動作する。
- **FR-113** 初回のユーザー操作(POWER ボタン)で AudioContext を開始する(iOS Safari の自動再生制限対応)。

### 3.2 SOUND — 音作り(FR-2xx)

- **FR-210** モノフォニックの減算シンセを 1 基提供する。音源は オシレーター(sawtooth / square / triangle / sine)と ホワイトノイズ の 2 系統ミックス。
- **FR-211** フィルタは LP / HP / BP の 3 種別から選択でき、**点灯中の種別をもう一度押すと OFF(真のバイパス)** になる(どのチップも消灯 = OFF)。LP/HP/BP はノイズ系打楽器の作り込みに必須。OFF はフィルター効果の A/B 比較用 — 実機は開き切りで代用するが、本機は CUTOFF 上限 12kHz のため専用のバイパス経路を持つ。
- **FR-212** 以下のパラメータをノブで編集できる。

| キー | 表示 | 範囲 | 役割 |
|---|---|---|---|
| pitchAmt | P.AMT | x1–x8 | ピッチエンベロープ量。発音瞬間の開始周波数 = 音程 × N。x1 で無効(通常演奏) |
| pitchTime | P.TIME | 5–500ms | ピッチ下降時間 |
| noise | NOISE | 0–100% | OSC とノイズの混合比 |
| cutoff | CUTOFF | 80–12kHz | フィルタカットオフ |
| reso | RESO | Q0.5–20 | レゾナンス |
| filterEnv | F.ENV | 0–4oct | フィルタエンベロープ量 |
| attack / decay / sustain / release | ADSR | — | 音量エンベロープ。sustain=0 でワンショット(打楽器) |

- **FR-213** キーボード UI(C2–C7、キー幅固定・あふれは横スクロール・初期位置は中央オクターブ)で演奏できる。タッチ・マウス両対応。PC キー割当: ホームロー(A W S E D F T G Y H U J K = C4–C5)。
- **FR-214** (廃止)オクターブ切替は C2–C7 描画+横スクロールに置き換えた。Z / X キーは FR-218 のトランスポーズに再割当。
- **FR-215** FX ラックを提供する。各エフェクトは個別に ON/OFF(バイパス)と MIX を持つ。

| FX | 実装 | パラメータ |
|---|---|---|
| DRIVE | WaveShaper(tanh) | drive, mix |
| AUTO-WAH | Filter + LFO | rate, depth, base, mix |
| DELAY | Delay + フィードバック Gain + ダンピング LPF | time, feedback, mix |
| REVERB | 並列コムフィルタ網(Delay + LPF + Gain) | size, damp, mix |

- **FR-216** FX は SOUND のボイス出力に直列で掛かる(マスターには掛からない)。
- **FR-217** リピート再生: 最後に弾いたノートを一定間隔でリトリガーするトグルを提供する(レート 1/4・1/8・1/16 = 0.5/0.25/0.125 秒、ゲートは間隔の 50%)。スケジューリングは Transport の look-ahead で行い(NFR-02)、リピート中に鍵盤を弾くとそのノートに追従する。タブ切替で停止する。
- **FR-218** トランスポーズ: 演奏ノートを ±12 半音の範囲で移調できる(REPEAT ドック内の −/+ ボタンおよび PC キー Z / X)。リピート中の変更は即座に発音へ反映され、ドックには実効ノート(移調後の音名)を表示する。

### 3.3 プリセット(FR-3xx)

- **FR-310** 現在の音(波形・フィルタ種別・全パラメータ・FX 状態)を名前付きプリセットとして保存・読込・削除できる。
- **FR-311** プリセット読込時はエンジンと UI(ノブ・選択状態)の両方に反映される。
- **FR-312** ファクトリープリセットを最低 6 種同梱する(808 系ドラム音を含む)。ファクトリープリセットは削除・上書き不可。
  - 現行: Kick / Tom / Snare / Closed Hat / Open Hat / Clap / Zap / Bass の 8 種
- **FR-313** プリセットを JSON としてエクスポート・インポートできる(スキーマは設計書 §6.1)。
- **FR-314** インポート時はスキーマバージョンと値域を検証し、不正データはエラー内容(キー名と許容範囲)を明示して拒否する。

### 3.4 PATTERN — シーケンサー(FR-4xx)

- **FR-410** 16 ステップ × 可変トラックのステップシーケンサーを提供する。トラックは追加・削除できる。
- **FR-411** 各トラックにサウンドプリセットを割り当てられる(ファクトリー / ユーザー保存の両方)。
- **FR-412** 各トラックはチューン(トリガー周波数 20–2000Hz)・ミュート・試聴を個別に持つ。
- **FR-413** トラックはチョークグループを持てる。同グループ内で後発音が先行音を止める(既定でハット 2 種が "hat" グループ)。
- **FR-414** BPM を 40〜220 の範囲で設定できる。
- **FR-415** スウィング(16 分裏の遅延量 0〜75%)を設定できる。
- **FR-416** 再生中のステップ位置(プレイヘッド)を UI 上に表示する。
- **FR-417** 再生中でもステップの ON/OFF・BPM・スウィング・トラック編集が反映される。
- **FR-418** パターンを名前付き保存・読込・削除でき、JSON 入出力できる(スキーマは設計書 §6.2)。
- **FR-419** パターンは各トラックに**プリセットのスナップショットを埋め込む**自己完結データとする。プリセットストアの後からの変更・削除はパターンに影響しない。エクスポートした JSON は単体で完全に再生可能。
- **FR-420** ファクトリーパターンを最低 2 種同梱する(削除不可)。

## 4. 非機能要件

- **NFR-01(レイテンシ)** 鍵盤/試聴の操作 → 発音開始 30ms 以内を目標とする。
- **NFR-02(タイミング精度)** ステップ再生は Tone.Transport による look-ahead スケジューリングで行い、UI スレッドの負荷でヨレないこと。`setTimeout` / `requestAnimationFrame` を発音タイミングの根拠にしない。プレイヘッド表示は `Tone.Draw` で音声時刻に同期する。
- **NFR-03(実装制約)** 音源・エフェクトは Tone.js の低レベルプリミティブのみで構成する。
  - 許可: `Oscillator` / `Noise` / `Filter` / `AmplitudeEnvelope` / `FrequencyEnvelope` / `Envelope` / `Gain` / `Volume` / `WaveShaper` / `Delay` / `Analyser` / `LFO` / `Transport` / `Frequency` などのユーティリティ
  - 禁止: `MembraneSynth` / `NoiseSynth` / `MetalSynth` / `PolySynth` / `Sampler` / `Player` / `Tone.Reverb` / `Tone.FeedbackDelay` / `Tone.Distortion` / `Tone.AutoWah` その他「完成品」シンセ・エフェクトクラス
- **NFR-04(可読性)** 音源・各エフェクトの実装は「回路図が読めるコード」を目標とする。ノード接続はコンストラクタ内に信号経路順で記述し、経路コメントを付す。
- **NFR-05(モバイル)** スマートフォン縦画面で全機能が操作可能。SOUND の 3 カラムは縦積みに折り返す。タップ対象は最小 44px 四方。
- **NFR-06(永続化の可搬性)** 保存層は抽象化し、Claude アーティファクト環境(window.storage)と通常 Web 環境(localStorage)の両方で動く実装を差し替え可能にする。
- **NFR-07(依存)** ランタイム依存は React + Tone.js のみ。
- **NFR-08(テスト容易性)** エンジンは UI から分離し、オフラインレンダリングによる自動検証が可能な構造とする。決定的ロジック(検証・往復・ストア)は node で、波形特性は `OfflineAudioContext` のある環境で実行する(設計書 §9)。
- **NFR-09(クリップ防止)** マスター段で出力が 0dBFS を超えないこと(ソフトクリップで保証。設計書 §1)。

## 5. 受け入れ基準

1. SOUND で 808 風のキックをゼロから作れる(sine + P.AMT + SUSTAIN 0)。ファクトリーの Kick を読み込んでも同等の音が鳴る
2. ノイズ + HP/BP フィルタでハット / スネア系の音が作れる
3. 作った音をプリセット保存 → リロード後に復元でき、読込で全ノブが追従する
4. エクスポートした JSON を別ブラウザでインポートして同じ音になる
5. PATTERN で任意のトラックに任意のプリセットを割り当てて 16 ステップが打ち込め、再生中の編集が反映される
6. ハット 2 トラックでチョークが効く(Open Hat 鳴動中に Closed Hat で停止)
7. エクスポートしたパターン JSON が、プリセットストアが空の環境でも単体で同じ演奏になる
8. スマートフォン実機で全操作ができ、体感遅延がない
9. 音源・FX コードを読んだ人が各信号経路を図に起こせる(レビュー観点)

## 6. 用語

| 用語 | 意味 |
|------|------|
| ピッチエンベロープ | 発音瞬間に周波数を高→低へ急降下させる制御。キック・タムの「胴鳴り」の正体 |
| サウンドプリセット | 波形 + フィルタ種別 + 全パラメータ + FX 状態の一式。SOUND タブで設計する単位 |
| トラック | パターン内の 1 行。プリセットのスナップショット + チューン + ステップ列 |
| チョークグループ | 同グループ内で後発音が先行音を強制停止する仕組み |
| パターン | 16 ステップ × 可変トラックの打ち込みデータ + BPM + スウィング。プリセット埋め込みで自己完結 |
