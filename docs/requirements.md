# LAB-1 ドラムモード 要件定義書

- ドキュメント種別: 要件定義書
- 対象: LAB-1(ブラウザ減算シンセ)への DRUM モード追加
- ステータス: Draft v1.0
- 作成日: 2026-07-12

---

## 1. 背景と目的

LAB-1 は減算合成の信号経路(OSC → FILTER → AMP ENV)を学ぶためのブラウザシンセである。
本プロジェクトでは、同じ合成プリミティブの組み合わせでドラム音(TR-808 系)が作れることを
示す DRUM モードを追加する。

### 1.1 プロジェクトゴール(優先順)

| # | ゴール | 含意 |
|---|--------|------|
| G1 | 合成の仕組みの学習 | 音源コードは信号経路が読める構造であること。ブラックボックス禁止 |
| G2 | 実際の曲作りで使える | 演奏レイテンシ・音質・保存機能が実用水準であること |
| G3 | OSS / ポートフォリオ公開 | 依存最小・コード可読性・ドキュメント整備。「プリミティブから808を合成する」を題材として明確に打ち出す |

### 1.2 スコープ外(明示)

- サンプル再生(Sampler / Player 系)。合成のみで音を作る
- MIDI 入出力(将来検討)
- オーディオファイル書き出し(将来検討。JSON エクスポートは対象内)
- ESP32 移植(本プロジェクトのゴールに含めない)

## 3. 機能要件

### 3.1 モード(FR-01x)

- **FR-010** LAB-1 に SYNTH / DRUM のモード切替を追加する。切替はワンタップ。
- **FR-011** モード切替時に再生中の音は自然に停止する(リリース経由。ブツ切りにしない)。
- **FR-012** 両モードは共通のマスター出力(音量・アナライザー)を共有する。オシロスコープは DRUM モードでも動作する。

### 3.2 ドラムボイス(FR-02x)

- **FR-020** 以下の 6 ボイス(808風構成)を実装する。

| ID | 名称 | 略号 |
|----|------|------|
| BD | バスドラム(Kick) | BD |
| SD | スネアドラム | SD |
| CH | クローズドハイハット | CH |
| OH | オープンハイハット | OH |
| CP | ハンドクラップ | CP |
| LT | ロータム | LT |

- **FR-021** 各ボイスは編集可能なパラメータ(3〜5個、設計書 §4 で定義)を持つ。
- **FR-022** CH / OH は同一チョークグループに属し、片方の発音がもう片方を止める(実機ハイハットの挙動)。
- **FR-023** 全ボイスは同時発音可能(6声ポリ相当)。同一ボイスの連打はリトリガー(前の発音を打ち切って再発音)。
- **FR-024** 各ボイスに個別レベル(音量)を持つ。

### 3.3 演奏(FR-03x)

- **FR-030** 6ボイスに対応するパッド UI を提供する。タッチ・マウス両対応、マルチタッチで同時叩き可能。
- **FR-031** PC キーボードにパッドを割り当てる(既定: Z X C V B N ないし数字列。設計時に確定)。
- **FR-032** パッド押下から発音までのレイテンシは体感即時であること(NFR-01 参照)。
- **FR-033** パッド押下時に視覚フィードバック(点灯)を返す。

### 3.4 音色編集とプリセット(FR-04x)

- **FR-040** パッド選択で該当ボイスのパラメータノブ群を表示し、リアルタイムに編集できる。
- **FR-041** 6ボイス分のパラメータ一式を「キット」として名前付き保存・読込・削除できる。
- **FR-042** ファクトリーキットを最低 2 種同梱する(例: 808風 / Techno系)。ファクトリーキットは削除不可。
- **FR-043** キットを JSON としてエクスポート・インポートできる(スキーマは設計書 §6)。
- **FR-044** インポート時はスキーマバージョンと値域を検証し、不正データはエラー内容を明示して拒否する。

### 3.5 シーケンサー(FR-05x)【Phase 2】

- **FR-050** 16ステップ × 6トラックのステップシーケンサーを提供する。
- **FR-051** BPM を 40〜220 の範囲で設定できる。
- **FR-052** スウィング(16分裏の遅延量 0〜75%)を設定できる。
- **FR-053** 再生中のステップ位置を UI 上に表示する。
- **FR-054** 再生中でもステップの ON/OFF・音色編集が反映される。
- **FR-055** パターンを名前付き保存・読込・削除でき、JSON 入出力できる(キットと同一の仕組み)。
- **FR-056** パターンはキット参照ではなく独立データとする(パターンとキットは自由に組み合わせられる)。

## 4. 非機能要件

- **NFR-01(レイテンシ)** パッド押下 → 発音開始 30ms 以内を目標とする(AudioContext の baseLatency + スケジューリング遅延込み)。
- **NFR-02(タイミング精度)** Phase 2 のステップ再生は Tone.Transport による look-ahead スケジューリングで行い、UI スレッドの負荷でヨレないこと。`setTimeout` / `requestAnimationFrame` を発音タイミングの根拠にしない。
- **NFR-03(実装制約)** 音源は Tone.js の低レベルプリミティブのみで構成する。

  - 許可: `Oscillator` / `Noise` / `Filter` / `AmplitudeEnvelope` / `FrequencyEnvelope` / `Envelope` / `Gain` / `Volume` / `WaveShaper` / `Analyser` / `LFO` / `Transport` / `Frequency` などのユーティリティ
  - 禁止: `MembraneSynth` / `NoiseSynth` / `MetalSynth` / `PolySynth` / `Sampler` / `Player` その他「完成品」シンセクラス

- **NFR-04(可読性)** 各ボイスの実装は「回路図が読めるコード」を目標とする。1ボイス = 1クラス、ノード接続はコンストラクタ内に信号経路順で記述し、経路コメントを付す。
- **NFR-05(モバイル)** スマートフォン縦画面で全機能が操作可能。パッドは指で押し分けられるサイズ(最小 44px 四方)。
- **NFR-06(永続化の可搬性)** 保存層は抽象化し、Claude アーティファクト環境(window.storage)と通常 Web 環境(localStorage)の両方で動く実装を差し替え可能にする(OSS 公開要件)。
- **NFR-07(依存)** ランタイム依存は React + Tone.js のみ。
- **NFR-08(テスト容易性)** 音源エンジンは UI から分離し、オフラインレンダリングによる自動検証が可能な構造とする(設計書 §9)。

## 5. 受け入れ基準(Phase 1)

1. パッド 6 個を叩き分けて 808 らしいキック / スネア / ハット / クラップ / タムが鳴る
2. CH 連打中に OH を叩く・その逆で、チョークが正しく効く
3. 任意に編集した音色をキット保存 → リロード後に復元できる
4. エクスポートした JSON を別ブラウザでインポートして同じ音になる
5. スマートフォン実機で 2 本指同時叩きができ、体感遅延がない
6. 音源コードを読んだ人が各ボイスの信号経路を図に起こせる(レビュー観点)

## 6. 用語

| 用語 | 意味 |
|------|------|
| ピッチエンベロープ | 発音瞬間に周波数を高→低へ急降下させる制御。キック・タムの「胴鳴り」の正体 |
| チョークグループ | 同グループ内で後発音が先行音を強制停止する仕組み |
| キット | 6ボイス分の音色パラメータ一式 |
| パターン | 16ステップ × 6トラックの打ち込みデータ + BPM + スウィング |
