# SOUND タブ レイアウト再設計 指示書(v2)

- ドキュメント種別: デザイン指示書(Claude Design 向け)
- 対象: **SOUND タブのレイアウト/情報設計のみ**。ビジュアルシステム(トークン・面の質感・ノブ/鍵盤/チップのスキン)は「LAB-1 Console 1b(as-built)」を**そのまま維持**する
- 対象外: PATTERN タブ、TOKENS、トップバー(変更なし)
- 作成日: 2026-07-14

---

## 1. 再設計の原則(優先順)

1. **同じ機能は同じ場所に** — 機能グループ(音源 / フィルター / FX / 演奏 / 管理)を跨いで部品を散らさない。現状は「PARAMS 10ノブ」に音源3+フィルター3+アンプ4が混在し、フィルターだけノブ(中央)とチップ(右)に割れている。これを解消する。
2. **実機・DAW の自然な配置** — パネルが上、鍵盤が下(現状維持。気に入っている)。セクション切替は Elektron 系グルーヴボックスのページボタン(SYN/FLTR/AMP/FX)を先例とする。演奏補助(REPEAT)は実機の ARP/HOLD の定位置=**鍵盤の左脇**。
3. **PC 表示で余白を使い切る** — 右レールと中央下部の空きを解消する。
4. **常時表示と呼び出し表示を分ける** — 音源の基本操作・鍵盤・モニター(SCOPE と SPECTRUM)・プリセットは常時。フィルターと FX の操作系はタブで呼び出す。
5. **信号経路の可読性(G1)を保つ** — タブを信号順 **OSC → FILTER → FX** に並べ、タブ列自体が経路図として読めるようにする。

## 2. 要素の分類(確定)

| 常時表示 | タブ呼び出し | 左カラム |
|---|---|---|
| SCOPE(マスター波形)+ SPECTRUM(フィルター応答重畳) | **FILTER**: LP/HP/BP チップ + CUTOFF/RESO/F.ENV | PRESETS 一式(現状のまま) |
| VOICE パネル(選択中タブの内容) | **FX**: チェーン表示 + 4 ユニット | |
| 鍵盤(フル幅)+ REPEAT | | |

- SPECTRUM は SCOPE と縦積みで**常設**(モニター列)。フィルター応答カーブも常時重畳するので、どのタブにいてもフィルターの効きが見える。FILTER タブは操作系(チップ+3ノブ)のみ。
- FX の折りたたみ(»/«)はタブ化に伴い**廃止**(タブがその役割を担う)。

## 3. レイアウト仕様(PC)

```
┌ TOPBAR: LAB-1 | SOUND/PATTERN | POWER ──────────────────────────────┐
├─────────┬────────────────────────────────────────────────┬──────────┤
│ PRESETS │  VOICE パネル(プレート)                        │ SCOPE    │
│ 200px   │  [OSC*] [FILTER] [FX]   ← 信号順セクションタブ │ SPECTRUM │
│ 縦一杯   │  ─ 選択タブのチップ+ノブ群(下記 §4)          │ (縦積み  │
│ フラット │                                                │  幅~280) │
│         ├────────────────────────────────────────────────┴──────────┤
│         │ [REPEAT] ██ 鍵盤 C2–C7 フル幅 ██████████████████████████ │
│         │  縦パネル    (あふれは横スクロール、初期中央)              │
└─────────┴───────────────────────────────────────────────────────────┘
```

- **2カラム構成**。従来の右レール(260px)は廃止し、操作系(チップ/FX)は VOICE パネルのタブへ、モニター(SPECTRUM)は右上のモニター列へ移す。
- **鍵盤は下段フル幅**(左カラムの右側全部)。C2–C7 = 白鍵36枚 × 44px は一般的なデスクトップ幅でほぼスクロールなしで収まる。キー幅は固定のまま、あふれたら横スクロール(既存挙動)。
- **SCOPE + SPECTRUM は右上に縦積みで常設**(モニター列、VOICE パネルと同じ行で高さを二分)。MASTER · PRE-CLIP / FILTER RESPONSE ラベル維持。緑=鳴っている音、橙=フィルター応答の色規範も維持。
- **REPEAT は鍵盤ウェルの左端に縦のミニパネル**として統合(⟳ トグル / レート 1/4・1/8・1/16 / 現在ノート表示)。ノブと鍵盤の間の独立行は廃止。

## 4. VOICE パネルのタブ内容

タブは信号順に並べ、経路として読めるよう `IN › OSC › FILTER › FX › OUT` の細い経路ストリップを添えてもよい(任意)。

| タブ | 内容 |
|---|---|
| **OSC**(既定) | WAVE チップ(sawtooth/square/triangle/sine)+ P.AMT / P.TIME / NOISE + ATTACK / DECAY / SUSTAIN / RELEASE(計7ノブ) |
| **FILTER** | LP/HP/BP チップ(点灯中を再クリックで OFF=バイパス、全消灯が OFF)+ CUTOFF / RESO / F.ENV(SPECTRUM は常設モニター側にあるためタブ内には置かない) |
| **FX** | チェーン表示(IN › DRIVE › WAH › DELAY › REV › OUT)+ 4 ユニット。横幅に余裕があるので横並び or 2×2 グリッドはデザイナー判断 |

**非選択タブにも状態を漏らす**(信号経路の可読性のため):

- FILTER タブのラベル脇に現在の種別(`LP` 等。OFF 時は `—`)を小さく表示
- FX タブのラベル脇に 4 ユニット分のミニ LED(ON のものだけ点灯)

## 5. 変更しないもの

- ビジュアルシステム一式: トークン、ウェル/プレート/ボタンの3値、ノブスキン(ティックリング+アーク+キャップ)、鍵盤の造形、チップ、LED、IBM Plex Mono
- トップバー、POWER(デフォルト通電)、左 PRESETS カラム(フラット構成)
- PATTERN タブ(別途フィードバック予定)
- 機能・挙動: ノブ操作モデル、touch-action、aria、フィルターのバイパストグル、REPEAT のノート追従

## 6. 技術制約(実装側の前提。design-brief.md §5 と同じ)

- 依存追加なし・外部 CDN 不可・セルフホスト
- ページ自体はスクロールしない(100dvh)。左カラムは内部スクロール。VOICE パネルはタブ化により通常は収まる想定(あふれる場合はパネル内スクロール)
- タブ選択は view state(localStorage 保持、既存の FX 折りたたみキーの置き換え)
- モバイル(≤820px)は縦積みに戻す。順序: 鍵盤(REPEAT 含む)→ VOICE パネル(タブ)→ SCOPE + SPECTRUM → PRESETS。タップ対象 44px 以上
